Module VarFamily

A variable family provides a minimal database for variables, with special support for various types.

In addition, a family has several "tiers" of these variables to allow rollback if the current set should become invalid.

Functions

VarFamily:AddToAutoPropagatedVars (group, name, target) Adds a variable to the auto-propagate list; such variables are automatically shadowed in all tiers up through target.
VarFamily:PropagateDownFrom (top) Replaces each lower tier with copies of the tier at top, cloning any non-raw, non-primitive variables.
VarFamily:PropagateUpTo (target) Replaces each tier from 2 to target with copies of the tier at 1 (the working set), cloning any non-raw, non-primitive variables.
VarFamily:RemoveFromAutoPropagatedVars (group, name) Removes a variable from the auto-propagation list.

Adds a variable to the auto-propagate list; such variables are automatically shadowed in all tiers up through target.

Parameters:

  • group string Variable group, which may be "bools", "nums", "raw", "delegates", "timers", or "timelines".
  • name Non-nil variable name.
  • target uint Highest tier in propagation.

VarFamily:GetVars (what)
Getter.

Parameters:

  • what string Variable type, one of "bools", "nums", "raw", "timers", "timelines", "delegates".

Returns:
